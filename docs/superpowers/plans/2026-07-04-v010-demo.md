# v0.1.0 Demo Packaging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Vibecoding Kit from a template MVP into a demonstrable v0.1.0 project with a runnable AI drift demo, README entry point, GitHub Actions CI, and release notes.

**Architecture:** Keep the existing shell-based kit intact. Add a self-contained `examples/ai-drift-demo/run-demo.sh` script that installs a temporary project, creates a locked task plan, simulates an unauthorized file change, proves the guard blocks it, then updates the plan and closes out. Add root documentation and CI metadata around that behavior.

**Tech Stack:** Bash, Git, Markdown, GitHub Actions, existing Vibecoding Kit installer and guard scripts.

---

## Task 2: Implement Runnable AI Drift Demo

**Files:**
- Create: `examples/ai-drift-demo/run-demo.sh`
- Create: `examples/ai-drift-demo/README.md`
- Create: `examples/ai-drift-demo/expected-output.txt`

- [x] **Step 1: Create demo runner**

The script must:

1. Install a temporary project using `installer/init.sh --profile agent-adapters`.
2. Create task artifacts for `T-001`.
3. Lock the plan.
4. Commit the task plan checkpoint.
5. Start `S-001`.
6. Add an allowed Java service file.
7. Add unauthorized `src/main/resources/application.yml`.
8. Run `plan-guard.sh` and require failure with unauthorized file output.
9. Update the plan to allow the config file, relock, and commit.
10. Run `plan-guard.sh`, commit implementation, complete the step, and write closeout report.

- [x] **Step 2: Create demo README and expected output**

The README should explain before/after:

- before: AI changes a file outside current step allowlist.
- guard: `PLAN_GUARD_FAIL: unauthorized file`.
- after: plan is updated, relocked, committed, and closeout report is written.

- [x] **Step 3: Run demo directly**

Run:

```bash
bash examples/ai-drift-demo/run-demo.sh
```

Expected output contains:

```txt
DEMO_PASS
```
