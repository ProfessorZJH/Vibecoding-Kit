# v0.5.0 - Governance Demos and UX Polish

Release date: 2026-07-06

## Summary

v0.5.0 adds executable governance demos that make Vibecoding Kit's core
behaviors visible from the repository root. The release focuses on demonstration
and onboarding polish, not new guard semantics.

## What's Included

- `examples/command-risk-demo/`: shows allow, require_approval, and block
  command classifications.
- `examples/risk-report-demo/`: shows HIGH risk evidence for runtime
  configuration changes.
- `examples/adapter-block-demo/`: shows managed adapter block updates preserving
  user-owned content.
- `examples/README.md`: lists all demos and their commands.
- README Demos section with all four 30-second demo commands.
- `scripts/test-kit.sh` coverage for demo presence, execution, and
  documentation references.

## Demo Commands

```bash
bash examples/ai-drift-demo/run-demo.sh
bash examples/command-risk-demo/run-demo.sh
bash examples/risk-report-demo/run-demo.sh
bash examples/adapter-block-demo/run-demo.sh
```

```bash
git diff --check

```

## Known Limits

- No CLI is included in this release.
- No YAML parser is added.
- No guard behavior changes are included.
- No automatic adapter sync is included.
- No demo depends on network access.
- Risk reports remain evidence for review; HIGH and CRITICAL risk are not
  standalone blockers in this release.
