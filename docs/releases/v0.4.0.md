# v0.4.0 - Managed Adapter Blocks MVP

Release date: 2026-07-05

## Summary

v0.4.0 adds managed adapter blocks and a safe block replacement script for AI
tool adapter files. Generated adapter content is now explicitly bounded by
Vibecoding Kit markers, so future updates can replace only kit-owned content and
preserve user-owned notes before and after the block.

## What's Included

- `core/scripts/adapter-block.sh`: validates and updates managed adapter blocks.
- `docs/adapter-managed-blocks.md`: marker convention and usage guide.
- Managed block markers in core `AGENTS.md` and `CLAUDE.md`.
- Managed block markers in Codex, Claude Code, Gemini, Cursor, Cline, Roo,
  Windsurf, and GitHub Copilot adapter files.
- `scripts/test-kit.sh` coverage for marker presence, installed output,
  `--check`, malformed marker failures, and `--update` preservation behavior.

## Managed Block Contract

```md
<!-- VIBECODING-KIT:BEGIN -->
managed adapter protocol content
<!-- VIBECODING-KIT:END -->
```

Content inside the markers is kit-owned. Content outside the markers is
user-owned. `adapter-block.sh --update` replaces only the marked block.

## Known Limits

- No CLI is included in this release.
- No global AI tool directories are synced.
- Missing markers are not inserted automatically.
- The script does not parse policy YAML or adapter templates.
- Managed blocks provide a safe update primitive; they do not decide when an
  adapter should be updated.
