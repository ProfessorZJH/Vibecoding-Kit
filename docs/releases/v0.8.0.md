# v0.8.0 - Real-World Java Governance Demo

Release date: 2026-07-06

## Summary

v0.8.0 adds a Java/Spring-style service governance demo. The demo shows how
Vibecoding Kit catches AI-assisted changes that drift outside a backend task
boundary, such as runtime configuration edits during a service-only task.

This release does not change core guard, risk report, closeout, policy,
adapter block, or installer semantics.

## What's Included

- `examples/java-service-governance-demo/` with a Java backend fixture.
- A service-only task boundary around
  `src/main/java/com/example/order/OrderService.java`.
- A simulated unauthorized edit to
  `src/main/resources/application.yml`.
- Plan guard evidence for the unauthorized file.
- Risk report evidence with `overall_risk: HIGH`.
- Closeout report creation for the audited demo state.
- `scripts/test-kit.sh` coverage for demo files, docs references, and stable
  output markers.

## Verification

Run from this kit repository:

```bash
git diff --check
bash -n scripts/test-kit.sh
bash -n scripts/readability-guard.sh
bash -n examples/java-service-governance-demo/run-demo.sh
bash examples/java-service-governance-demo/run-demo.sh
bash scripts/readability-guard.sh
bash scripts/test-kit.sh
```

## Known Limits

- The demo does not run Maven, Gradle, or Java compilation.
- The demo does not require network access.
- The demo does not add a CLI.
- The demo does not change installer behavior.
- The demo does not change command guard, risk report, drift guard, closeout,
  adapter block, or policy behavior.
- HIGH and CRITICAL risk reports remain evidence for review; they are not
  standalone blockers in this release.
