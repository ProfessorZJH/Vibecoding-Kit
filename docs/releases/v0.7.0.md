# v0.7.0 - Installer UX Hardening

Release date: 2026-07-06

## Summary

v0.7.0 hardens the shell installer with dry-run output, generated project
health checks, and repeat-install safety. The release improves first-use trust:
users can preview what would be written, verify a generated target project, and
rerun the installer without silently overwriting local edits.

This release keeps the project shell-first. It does not add a CLI wrapper or
change guard, risk, closeout, adapter block, or policy semantics.

## What's Included

- `installer/init.sh --dry-run` previews the target, project name, selected
  profiles, selected CI workflows, files that would be created, identical files
  that would be skipped, and conflicts.
- `core/scripts/ai-doctor.sh` is installed into generated target projects as
  `scripts/ai-doctor.sh`.
- `scripts/ai-doctor.sh` verifies core generated docs and governance scripts
  from inside the target project.
- Repeat installer runs skip identical files and fail with
  `VIBECODING_KIT_INSTALL_CONFLICT` when an existing target file has different
  content.
- `scripts/test-kit.sh` covers dry-run behavior, generated doctor checks, clean
  repeat installs, and preservation of user-modified conflicting files.
- README documents installer dry-run and doctor verification commands.

## Installer Verification

Preview an install without creating the target:

```bash
bash installer/init.sh \
  --dry-run \
  --target /tmp/vibecoding-kit-dry-run \
  --name dry-run-demo \
  --profile agent-adapters \
  --ci none
```

Install and verify from inside the target:

```bash
bash installer/init.sh \
  --target /tmp/vibecoding-kit-demo \
  --name installer-demo \
  --profile agent-adapters \
  --ci none

cd /tmp/vibecoding-kit-demo
bash scripts/ai-doctor.sh
```

## Conflict Behavior

The installer compares the full staged payload with the target project before
writing files:

- missing files are created
- identical existing files are skipped
- different existing files are reported as conflicts and preserved

Conflicts exit non-zero and print:

```text
VIBECODING_KIT_INSTALL_CONFLICT
will_conflict:
  AGENTS.md
```

## Known Limits

- No TypeScript, Node, Python, or compiled CLI is included.
- No YAML parser is added.
- No automatic uninstall, rollback, or backup workflow is included.
- No global AI tool directory sync is included.
- No guard, risk report, closeout, adapter block, or policy behavior changes
  are included.
- No network-dependent installer behavior is added.
- HIGH and CRITICAL risk reports remain evidence for review; they are not
  standalone blockers in this release.
