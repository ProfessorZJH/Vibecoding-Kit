# v0.2.0

Release date: 2026-07-05

## Summary

v0.2.0 upgrades Vibecoding Kit from a plan-drift guard template into an AI
agent governance MVP. The repository now combines modular prompts, documented
policies, shell guards, command classification, risk reporting, and closeout
evidence into one repository-level workflow.

The core operating rule stays the same:

> Prompt guides behavior. Policy defines boundaries. Guard checks violations.
> Report preserves evidence.

## What's Included

- Modular prompt layer under `core/prompts/` for explore, plan, implement,
  command classification, security review, closeout, and memory summary.
- Policy layer under `core/docs/policies/` for path sensitivity, command
  classes, and file-change risk levels.
- `scripts/command-guard.sh` as a classifier-only command risk guard with
  `allow`, `require_approval`, and `block` outcomes.
- `scripts/risk-report.sh` to generate markdown and JSON risk evidence for the
  current task.
- `scripts/drift-guard.sh` integration that generates risk reports when task
  context exists.
- `scripts/task-closeout.sh` integration that records risk report artifacts and
  summary decisions in closeout evidence.
- Expanded `scripts/test-kit.sh` coverage for prompt installation, policy
  installation, command classification, risk reporting, drift integration, and
  closeout evidence.

## Verification

Run from this kit repository:

```bash
bash scripts/test-kit.sh
git diff --check
```

Representative governance commands in generated projects:

```bash
bash scripts/command-guard.sh "git status"
bash scripts/command-guard.sh "npm install"
bash scripts/risk-report.sh T-xxx
bash scripts/drift-guard.sh
bash scripts/task-closeout.sh T-xxx --write-report
```

## Known Limits

- Policy files are documentation-first in this release. Shell guards still use
  embedded pattern checks instead of parsing YAML policies.
- HIGH and CRITICAL file risk classifications are report-only in v0.2.0. They
  do not act as standalone blockers by themselves.
- `command-guard.sh` classifies commands but does not execute them, approve
  them, or sandbox them.
- Human review, semantic tests, and product judgment are still required.
