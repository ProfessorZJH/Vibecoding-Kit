# v0.3.0 - Workflow and Adapter Capability MVP

Release date: 2026-07-05

## Summary

v0.3.0 adds a reusable workflow layer and adapter capability matrix on top of
the existing prompt, policy, guard, and report layers. AI coding tools can now
follow common project-scan, task-create, plan-lock, implement-step,
risk-review, and closeout workflows while using repository files as the source
of truth.

## What's Included

- `core/workflows/README.md`: workflow index for generated projects.
- `core/workflows/project-scan.md`: read-only repository discovery workflow.
- `core/workflows/task-create.md`: task, requirements, design, and plan creation
  workflow.
- `core/workflows/plan-lock.md`: plan validation and lock workflow.
- `core/workflows/implement-step.md`: current-step implementation workflow.
- `core/workflows/risk-review.md`: command and file-change risk evidence
  workflow.
- `core/workflows/closeout.md`: final guard, report, and checkpoint workflow.
- `docs/adapter-capabilities.md`: tool capability matrix for Codex, Claude
  Code, Gemini, Cursor, Cline, Roo, Windsurf, GitHub Copilot, and Superpowers.
- Agent adapter entry files now reference `workflows/README.md`.
- `scripts/test-kit.sh` verifies workflow files, installed workflow files,
  adapter references, and adapter capability documentation.

## Architecture

```text
Prompt -> Workflow -> Policy -> Guard -> Report
```

Prompt modules guide behavior. Workflow documents sequence the task lifecycle.
Policy files define boundaries. Guard scripts check violations. Reports preserve
evidence.

## Known Limits

- Workflow documents are documentation-only in v0.3.0.
- Adapter files point tools to the workflow layer, but enforcement still comes
  from shell guards, hooks, CI, and human review.
- Managed adapter block updates are reserved for a later release.
