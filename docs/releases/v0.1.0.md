# v0.1.0

Release date: 2026-07-04

## Summary

v0.1.0 packages Vibecoding Kit as a demonstrable AI development workflow tool.
The core Plan Engine was already present; this release adds the missing product
surface: a runnable drift demo, README entry point, CI workflow, and changelog.

## What's Included

- `examples/ai-drift-demo/run-demo.sh`: installs a temporary project, locks a
  task plan, simulates an AI-style unauthorized configuration edit, verifies
  `PLAN_GUARD_FAIL`, updates the task card and plan, relocks, and writes a
  closeout report.
- `examples/ai-drift-demo/README.md`: explains the before/after flow.
- `.github/workflows/test-kit.yml`: runs the kit integration tests on GitHub.
- `CHANGELOG.md`: records release-level changes.
- README demo section: shows the failure signal users should expect.

## Demo Contract

```bash
bash examples/ai-drift-demo/run-demo.sh
```

Expected signal:

```txt
DEMO_STEP unauthorized_change_blocked
PLAN_GUARD_FAIL: unauthorized file
src/main/resources/application.yml
DEMO_STEP relocked_after_plan_update
DEMO_STEP closeout_report_written
DEMO_PASS
```

## Known Limits

- The CLI is still shell-first. A TypeScript or Go wrapper can be added later
  without replacing the existing scripts.
- The demo uses a synthetic Java service, not a full production Spring/Vue app.
- The guards block repository-visible drift; they do not replace code review,
  semantic tests, or human product judgment.
