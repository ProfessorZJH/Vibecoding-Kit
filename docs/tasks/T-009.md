# T-009 Release Quality Gate and Readability

## Goal

Close the v0.7.0 release quality gap by making the public raw files readable
and adding a lightweight guard that prevents key files from being compressed
back into a few extremely long lines.

## Background

T-008 completed installer dry-run behavior, generated-project doctor coverage,
repeat-install safety, and installer UX regression coverage. The release branch
is aligned with `master` and tagged as `v0.7.0`.

The remaining issue is presentation and maintainability. Public docs and shell
files must be readable in raw GitHub views, review diffs, and terminal editors.
This task does not add product behavior; it only reflows existing text/scripts
and wires a readability quality gate into the existing test kit.
