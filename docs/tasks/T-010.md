# T-010 Java Service Governance Demo

## Goal

Add a Java/Spring-style backend demo that shows Vibecoding Kit detecting AI
changes outside a service-layer task boundary.

## Background

T-009 added the release readability gate. The kit now has demos for drift,
command risk, risk reports, managed adapter blocks, generated projects,
installer UX, and readability coverage.

T-010 adds a more realistic backend scenario: a task allows changing only an
order service file, while an AI-assisted edit also changes runtime
configuration. The demo proves the existing guard, risk report, and closeout
flow can surface that drift without changing core script semantics.
