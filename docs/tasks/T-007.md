# T-007 Generated Project Smoke Demo

## Goal

Add a short generated-project demo that proves Vibecoding Kit can be installed
into a clean target repository and run the basic governance loop inside that
generated project.

## Background

v0.5.0 added repository-level governance demos for plan drift, command risk,
risk reporting, and managed adapter blocks. Those demos prove the kit's
individual capabilities from this repository.

T-007 adds the next onboarding proof: a user can generate a fresh target
project with `installer/init.sh`, then run the installed preflight, drift/risk,
and closeout scripts from inside that target project. This is a smoke demo, not
a new runtime system.

## Required Work

- Add `examples/generated-project-demo/` with README, executable `run-demo.sh`,
  and `expected-output.txt`.
- The demo must create a temporary target project and run
  `installer/init.sh`.
- The demo must verify generated files in the target project, including
  `docs/AI_STATE.yml`, `scripts/ai-preflight.sh`, `scripts/drift-guard.sh`,
  `scripts/risk-report.sh`, and `scripts/task-closeout.sh`.
- The demo must run preflight inside the target project.
- The demo must run drift guard inside the target project, which also writes the
  initial risk report.
- The demo must commit the generated project snapshot so closeout can report a
  commit checkpoint.
- The demo must run `scripts/task-closeout.sh T-000 --no-tests --write-report`
  inside the target project and verify the closeout report exists.
- Update README and `examples/README.md` so users can find the generated
  project demo.
- Extend `scripts/test-kit.sh` so the demo and documentation references stay
  covered.
- Add `docs/releases/v0.6.0.md` documenting the generated project smoke demo
  scope and limits.
