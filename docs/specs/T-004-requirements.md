# T-004 Requirements

status: approved

## Problem

Adapter files are generated by Vibecoding Kit, but they live in normal project
locations where users may add local notes. Reinstalling or updating the kit must
not overwrite those user-owned edits. The project needs a minimal safe-update
primitive before adding broader adapter sync or CLI features.

## Goals

- Define a managed block convention for generated adapter content.
- Add a shell script that validates managed block structure.
- Add a shell script mode that replaces only the managed block from a template.
- Keep user content before and after the managed block unchanged.
- Add managed blocks to core and profile adapter entry files.
- Add test-kit coverage for script behavior, adapter markers, and installed
  target output.
- Document the convention and release boundary.

## Non-Goals

- Do not add a CLI.
- Do not add global tool configuration sync.
- Do not insert markers into arbitrary files automatically.
- Do not overwrite user-owned content outside the managed block.
- Do not add a template language or variable substitution.
- Do not parse policy YAML.
- Do not change existing guard, report, or installer semantics beyond copying
  the new script as part of `core/scripts`.

## Acceptance Criteria

- `core/scripts/adapter-block.sh` supports:
  - `--check <adapter-file>`
  - `--update <adapter-file> <template-file>`
- `--check` exits 0 for exactly one begin marker, exactly one end marker, and
  begin before end.
- `--check` exits non-zero for missing markers, duplicate begin markers,
  duplicate end markers, or end-before-begin ordering.
- `--update` validates both target and template before writing.
- `--update` replaces the target block from begin marker through end marker with
  the template block from begin marker through end marker.
- `--update` preserves all target content before the begin marker and after the
  end marker.
- Adapter files in `core/` and `profiles/agent-adapters/root/` contain exactly
  one managed block.
- Installed projects include the managed block markers in generated adapter
  files.
- `scripts/test-kit.sh` verifies the behavior without requiring a global install
  or network access.

## Risk

medium
